2. AI Models — Required & Optional

PAN uses a tiered AI architecture. Models are pulled automatically on first run.

2a. Required Local Models (Ollama)

These are downloaded automatically. You must have both.

ModelParamsDownloadRAMWhat It Does
qwen3:4b Required4B ~3 GB~3.5 GB Local inference engine. Handles sensitivity classification ("is this safe to send to cloud?"), privacy routing, content security (Guardian), and serves as fallback when cloud is unavailable. Leads benchmarks in the 4B class. Runs on CPU or GPU.
qwen3-embedding:0.6b Required0.6B ~0.5 GB~0.8 GB Vector embedding engine. Converts text into 1024-dimension number vectors for semantic search. When you say "remind me about the car thing," this is what finds the matching memory. Supports 100+ languages. Runs silently in background.

2b. Optional Local Models (Upgrade)

ModelParamsDownloadRAMWhen to Use
qwen3:8b8B ~5 GB~6 GB Upgrade for 32 GB RAM machines. Best-in-class 8B — outperforms most 12B models. Better local classification + sensitivity routing. Cannot run on 16 GB RAM.
qwen3:32b32B ~20 GB~22 GB 64 GB RAM only. Can replace Cerebras for Dream, Archivist, Scout — run fully offline. Needs 16+ GB VRAM for decent speed.
gemma3:4b4B ~3.3 GB~3.5 GB Alternative to qwen3:4b. Viable but outperformed by Qwen 3 and Phi-4 on benchmarks.
phi-43.8B ~2.3 GB~3 GB Microsoft. Strong reasoning for size. Alternative to qwen3:4b.
1B models are NOT sufficient. They fail on multi-step reasoning and produce unreliable sensitivity classifications. PAN requires minimum 4B parameters for local inference. The embedding model (0.6B) is fine because embeddings are a simpler task than reasoning.

2c. Cloud Models

These run on remote servers. PAN sends the query, gets a response. Your data goes through the sensitivity router first.

ModelProviderCostSpeedWhat It DoesCould a Smaller Model Do It?
Qwen 3 235B Cerebras Free Tier $0 (1M tokens/day) ~580ms Primary reasoning backbone. Powers Dream, Archivist, Scout, Orchestrator, Evolution, Intuition classification. 120B: Yes, for most services. 70B: Dream and Evolution would degrade. 8B: Not enough for consolidation tasks.
GPT-OSS 120B Cerebras Free Tier $0 (1M tokens/day) ~330ms Alternative backbone. Faster than 235B. Can handle Scout, Guardian, Archivist. Slightly weaker on complex reasoning. Good enough for: Scout, Guardian, Orchestrator. Weaker on: Dream cycle, Evolution (needs deeper reasoning).
LLaMA 3.1 8B Cerebras Free Tier $0 (1M tokens/day) ~200ms Fast classifier. Powers Guardian (content security). Fastest option. This IS the smallest viable cloud model. Below 8B, classification becomes unreliable.
Free tier limits (per model): 1M tokens/day, 60K tokens/min, 30 requests/min. Background services currently consume ~200K tokens/day with phone OFF. With phone active: ~300-400K/day. Well within free tier.
Cerebras Free Tier Risk: Free today, but paid tiers exist ($0.10–0.60/M tokens). At current usage (200K/day), going paid would cost ~$5–8/month. Heavy usage (2M/day): ~$54/month. PAN is designed to swap backends (Groq, Together, local 32B) when needed.

4. System Requirements

16 GB RAM — Minimum

  • OS: Windows 10/11, Ubuntu 20.04+, macOS 12+
  • RAM: 16 GB (8 GB free for PAN + models)
  • Disk: ~1.5 GB
  • GPU: Not required (CPU inference works, slower)
  • Network: Required (Cerebras + WireGuard)
  • Local models: qwen3:4b + qwen3-embedding:0.6b
  • Cloud: Cerebras free tier handles Dream, Archivist, Scout, etc.

Cannot run 8B models. All heavy reasoning goes to cloud. Local model handles privacy-sensitive tasks and fallback only.

32 GB RAM — Recommended

  • RAM: 32 GB (16 GB free)
  • Disk: ~4 GB
  • GPU: NVIDIA (CUDA) or AMD (ROCm) for 10-20x faster inference
  • Local models: qwen3:8b (inference) + qwen3-embedding:0.6b
  • Cloud: Still uses Cerebras for Dream/Evolution (needs 70B+)

Can run 8B locally for better sensitivity routing, Guardian, and Intuition fallback. Dream and Evolution still need cloud.

64 GB RAM — Fully Offline Capable

  • Disk: ~25+ GB (all models)
  • GPU: 16+ GB VRAM recommended (RTX 4070+, RX 7800+)
  • Local model: qwen3:32b (~20 GB) — this is what replaces Cerebras for Dream, Archivist, Orchestrator, Evolution
  • Everything runs locally. No cloud dependency at all. Full privacy.
  • Claude CLI: Subscription — for interactive terminal + Forge + PAN automation
  • Whisper + Kokoro: Full local voice pipeline, no Google STT needed

qwen3:32b at 32B parameters can handle Dream (needs 70B+ ideally, but 32B is functional), Archivist (30B+ works), Orchestrator (30B+ works), and Evolution (reduced quality vs 70B+). For truly equivalent quality to Cerebras 235B, you'd need qwen3:72b (~45 GB, 64+ GB RAM, 24+ GB VRAM).

5. What Runs Locally vs. Cloud

FunctionWhereWhy
Vector embeddings Always Local Embeddings encode your data into searchable vectors. They contain the meaning of everything you've said. Must never leave your machine.
Sensitivity classification Always Local Runs BEFORE any cloud call. Decides: is this password-level sensitive (local only)? Or safe to send to Cerebras?
Level 3 (critical) queries Always Local Passwords, encryption keys, medical records. Zero bytes leave your machine. Processed by qwen3:4b locally.
Privacy compute Always Local Adds differential privacy noise, strips PII, anonymizes data BEFORE anything goes to cloud.
TTS (voice output) Always Local Kokoro-82M runs on your device. Your voice responses never touch a cloud server.
Voice routing Cloud Needs fast, high-quality reasoning. Cerebras Qwen 235B (free) or Claude Haiku.
Dream / Archivist / Evolution Cloud Complex reasoning over large context windows. Needs 30-70B+ parameters. Runs every 6h on Cerebras (free).
Scout / Orchestrator Cloud Background intelligence services. Use Cerebras free tier. ~200K tokens/day total.
Forge (autodev) CLI Spawns Claude sessions for automated coding. Also controls PAN: opens windows, manages devices, orchestrates tasks.
STT (voice input) Local / Cloud Desktop: Whisper (local, ~145 MB). Phone: Google STT (on-device, no cloud in most cases).

8. Installation

Windows

Download the installer. Run it. Done.

The installer handles everything automatically:

  • Downloads and bundles Node.js (you never see it)
  • Installs PAN service + all dependencies
  • Generates encryption keys
  • Creates desktop + Start Menu shortcuts
  • Registers PAN as a startup service (auto-starts with Windows)
  • Opens the Setup Wizard in PAN's own window
  • Setup Wizard: enter your name → choose AI backend → connect phone via WireGuard → done
  • AI models download in the background on first run

Linux / macOS

Download the package. Install it. Done.

Or from terminal: curl -sSL https://pan.dev/install | bash

  • Detects your architecture (x64/arm64) automatically
  • Installs to ~/.local/share/pan/
  • Creates systemd service (Linux) or launchd plist (macOS)
  • Creates desktop shortcut
  • Opens Setup Wizard → same flow as Windows

Docker (Advanced)

docker compose up -d — 685 MB image, data persisted in named volume. Ollama runs on host or as separate container.

What happens after install: PAN starts, checks if Ollama is installed, starts it if needed, pulls qwen3-embedding:0.6b (~500 MB) and qwen3:4b (~3 GB) in the background. You can use PAN immediately — cloud services work while local models download. Full local capability is ready in 5–15 minutes depending on your internet speed.
