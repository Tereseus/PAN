4. System Requirements

Minimum — 16 GB RAM

  • OS: Windows 10/11, Ubuntu 20.04+, macOS 12+
  • RAM: 16 GB (8 GB free for PAN + model)
  • Disk: ~2 GB (PAN + Node + Ollama CPU-only + gemma3:4b)
  • GPU: Not required (CPU inference works)
  • Network: Required for Cerebras cloud services + Tailscale
  • Local model: qwen3:4b (~3 GB download, ~3.5 GB RAM)
  • Embedding model: qwen3-embedding:0.6b (0.5 GB, saves RAM)

Total disk: ~2 GB. Reasoning services use Cerebras cloud (free). Local model handles embeddings + privacy-sensitive queries only.

Recommended — 32 GB RAM

  • OS: Windows 10/11, Ubuntu 20.04+, macOS 12+
  • RAM: 32 GB (16 GB free)
  • Disk: ~5 GB (PAN + Ollama + llama3.2 + gemma3:4b)
  • GPU: NVIDIA (CUDA) or AMD (ROCm) for faster inference
  • Network: Required for cloud + Tailscale
  • Local models: llama3.2 (embeddings) + qwen3:4b (inference)

Total disk: ~5 GB. Full local privacy pipeline. Both embedding + inference models loaded. GPU enables 10-20x faster local responses.

Full — 64 GB RAM (Power User / Developer)

  • Disk: ~8+ GB (all models + Whisper + Claude CLI)
  • GPU: 8+ GB VRAM recommended (RTX 3060+, RX 7600+)
  • Everything local: Can run without any cloud dependency
  • Claude CLI: Interactive terminal sessions ($100/mo subscription)
  • Whisper: Local speech-to-text (faster-whisper-base, ~145 MB)
  • Multiple models: Hot-swap between gemma3, qwen3, llama3.2

5. What Runs Locally vs. Cloud

FunctionLocationWhy
Vector embeddings Always Local Embeddings encode user data. Must never leave the machine.
Sensitivity classification Always Local Determines if content is safe to send to cloud. Must run locally first.
Level 3 (critical) queries Always Local Passwords, keys, medical records. Zero cloud exposure.
Privacy compute Always Local Differential privacy, data anonymization before any cloud call.
Voice routing / classification Cloud Needs reasoning quality. Cerebras Qwen 235B (free) or Claude Haiku.
Dream cycle / memory consolidation Cloud Complex reasoning over large context. Runs every 6h via Cerebras (free).
Scout / Orchestrator / Evolution Cloud Background services needing 30B+ reasoning. All use free Cerebras tier.
Interactive terminal (Forge) CLI Runs Claude Haiku via CLI. Requires paid subscription.
STT (speech-to-text) Local / Cloud Whisper (local) on desktop, Google STT (cloud) on phone.
Cerebras Free Tier Risk: Currently free (1M tokens/day, 30 req/min, 8K context). No sunset date announced, but paid tiers exist ($0.10–0.60/M tokens, or $50–200/mo plans). The free tier is clearly a funnel to paid. Treat as "free until it isn't." PAN should be designed to swap backends (Groq, Together, local) when this changes. Estimated cost if Cerebras goes paid: ~$20/mo for PAN's background service usage.

7. Model Comparison — Which to Ship

ModelParamsSizeRAMQualitySpeed (CPU)Verdict
qwen3:4b4B ~3 GB~3.5 GB Leads 4B class on benchmarks. Strong multilingual + reasoning. ~2–4 tok/s Ship this as default
gemma3:4b4B 3.3 GB~3.5 GB Decent 4B model. Does NOT match 12B quality despite marketing. ~2–4 tok/s Viable fallback. Outperformed by Qwen 3 and Phi-4.
llama3.2:7b7B 4.7 GB~5.5 GB Good general purpose, best embeddings (3072D). Leads on coding (72.6% HumanEval). ~1–2 tok/s Keep for embeddings; replace with qwen3-embedding for size
qwen3:8b8B ~5 GB~6 GB Best-in-class 8B. Outperforms all 4B and most 12B models. ~1–2 tok/s Upgrade pick for 32 GB RAM machines

gemma3:1b1B 0.8 GB~1.2 GB Weak — fails multi-step tasks ~8–12 tok/s NOT viable for PAN classification
Recommendation: Ship qwen3:4b as the default local model. It leads benchmarks in the 4B class, fits in 16 GB RAM alongside OS + PAN server. For 32 GB machines, offer qwen3:8b as an upgrade. Replace llama3.2 embeddings with qwen3-embedding:0.6b (0.5 GB, 1024D, 100+ languages) to save ~4 GB.

GPU VRAM requirements: gemma3:4b needs ~3 GB VRAM for full GPU offload. Cards with 4+ GB VRAM (GTX 1650+, RX 5500+) can fully offload. Cards with less use partial offload (some layers on GPU, rest on CPU — still faster than pure CPU).

9. Tiered Install Options

Lite Install (~1.5 GB)

For 16 GB RAM, no GPU, minimal disk

  • PAN service + portable Node.js
  • Ollama (CPU-only runtime)
  • qwen3:4b (local inference)
  • qwen3-embedding:0.6b (tiny embedding model)
  • Tailscale
  • Cloud reasoning via Cerebras (free)

Full Install (~5 GB)

For 32 GB RAM with discrete GPU

  • Everything in Lite, plus:
  • Ollama with GPU runtime (CUDA/ROCm)
  • llama3.2 (full 3072D embeddings)
  • qwen3:4b (local inference)
  • Claude CLI (optional, prompted)
  • Whisper STT (local speech-to-text)
  • Tailscale

11. Installation Flow

Windows

  1. User runs: irm https://get.pan.dev/install.ps1 | iex (or downloads .exe)
  2. Installer downloads portable Node.js 22.16.0 (~32 MB)
  3. Extracts PAN service, runs npm install with portable Node
  4. Generates encryption keys (pan.key, audit.key)
  5. Creates PAN.bat (crash-restart loop) + desktop/Start Menu shortcuts
  6. Registers as Windows service (admin) or Startup folder (non-admin)
  7. Opens firewall for port 7777
  8. Launches PAN → Setup Wizard opens at http://localhost:7777/setup
  9. Setup Wizard: Name → AI config → Tailscale → redirects to dashboard
  10. First-run background: Ollama check → model pull (qwen3:4b, ~3 GB) → ready

Linux

  1. User runs: curl -sSL https://pan.dev/install | bash
  2. Installer downloads portable Node.js for detected arch (x64/arm64)
  3. Extracts to ~/.local/share/pan/
  4. Runs npm install, generates encryption keys
  5. Creates systemd user service (Linux) or launchd plist (macOS)
  6. Creates desktop shortcut via XDG .desktop file
  7. Symlinks pan to ~/.local/bin/
  8. Starts PAN → Setup Wizard at http://localhost:7777/setup

Docker

  1. docker compose up -d
  2. Image builds via multi-stage Dockerfile (~685 MB final)
  3. Data persisted in named volume
  4. Ollama runs on host or as separate container
