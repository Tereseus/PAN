

# Session Restart Context

**On service restart:**
1. Always read the current conversation first to understand what was being worked on
2. Check the last 3-5 messages to see if an issue is actively being debugged
3. Load memory files second, use them as background reference only
4. If memory contradicts conversation, conversation wins
5. Report what was being worked on: "Last working on: [specific task] — [status]"

Conversation is source-of-truth; memory is secondary.
